Network Application Architectures

  • Application architecture
    • Designed by appication developer
    • Dictates how application is structured over various end systems
    • Usually draws on either client-server or peer-to-peer
  • Client-server architecture
    • Always-on host (server) fields requests from many other hosts (clients)
    • Web server recieves request from client host, responds with the requested object
    • Clients do not directly communicate with each other
    • Server has a fixed, well-known IP address
    • EX: World Wide Web, FTP, Telnet, e-mail
    • If a single host server is insufficient, maby connected servers can be used as a data center
    • Service providers must pay recurring interconnection and bandwith costs for sending data from data centers
  • Peer-to-Peer (P2P) architecture
    • Minimal to no reliance on dedicated servers
    • Direct communication between pairs of intermittently connected hosts, peers
    • Peers are not owned by service provider, but the users
    • Ex: BitTorrent, Xunlei, Skype, Kankan, PPstream
    • Self-scaling: each peer adds both workload and capacity
    • Cost effective: don't require significant server infrastructure or bandwith
    • Not ISP friendly: residential ISPs are made for more downstream than upstream traffic, and P2P adds heavy upstream usage
    • Difficult to secure by nature
    • Require an incentive for users to volunteer their resources to the network
