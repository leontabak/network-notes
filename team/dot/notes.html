Class notes 10/7/2016

  • Finite State Machine
    • An abstract machine which can be in one of a finite number of states
    • Any given finite state machine described by:
      1. A finite set of states
      2. Alphabet - a finite set of symbols
      3. Transition function
        • A set of rules (can be given in a table)
        • If in state [A] and looking at symbol [*], next state is [B]
      4. Starting state
      5. Set of accepting states
    • Language: set of symbols and rules to distinguish legal from illegal (describes a machine)
    • Machine: set of rules which recognize a language
    • Lexx
      • Unix program
      • Given a description of a regular language, produces a C program (finite state machine) that recognizes the language
  • Regular Expressions
    • Search for more complicated patterns than exact matches
  • 4 Types of Languages
    • Regular Languages
      • Simplest in heirarchy
  • 4 kinds of codes
    1. Convenience: unicode/ASCII
    2. Data compression: ZIP, MP3, JPEG
    3. Error detection and correction
    4. Encryption

Class Notes 10/10/2016

  • Different forms of identifier are used in different circumstances
  • Hostname
    • variable-length alphanumeric string identifier
    • provide little information about the sit'es location
    • difficult for computers to parse
  • IPv4 Address
    • 4-byte computer-readable numeric identifier
    • describes its location
    • Format: byte.byte.byte.byte (123.45.67.89)
  • Mnemonic: device to help you memorize something
  • Domain Name System
    • Two part definition
      1. Distributed database
      2. API for making queries on said database
    • Popular example: Berkeley Internet Name Domain
    • DNS runs on port 53 over UDP
    • Application-level protocol used by other application-level protocols
    • Process of using the DNS to fetch an IP address adds delay (from milliseconds to seconds
    • To reduce delay, nearby DNS server caches frequently requested IPs
    • Cache: local memory that stores what was looked up last
    • Specified in RFC 1034 and RFC 1035, updated in several more
    • Users do not interact directly with the DNS, only applications do
  • DNS Services
    • Host Aliasing
      • Host with a complicated hostname can have one or more alias names
      • Complicated base url is the canonical hostname
      • Alias names are usually more mnemonic
    • Mail Server Aliasing
      • Translate mnemonic e-mail address endings (@hotmail.com, @gmail.com, etc.)
      • Invoked by mail application
      • MX record permits company's mail server and web server to have identical alias names
    • Load Distribution
      • Busy sites have IPs to alleviate strain on the system
      • DNS alternates between which IP it distributes first to prevent all traffic from going to the same IP
  • Simple centralized DNS has many problems
    • Single point of failure: if the server fails, the entire internet crashes
    • Traffic volume would be immense, creating a long queue delay
    • Anyone far from the database would have high delays
    • High maintenance requirements
  • Distributed DNS
    • No single machine contains all of the mappings
    • Machines are distributed across the world
    • Hierarchy of servers
      1. Root server
      2. TLD server
      3. Authoritative server
  • DNS Message Format
    1. Identification/Flags (4 bytes each)
      • Identification: 16-bit number identifying the query
      • Flags: Yes/no indicators defining the kind of query
    2. Number of questions/Number of answer RRs (4 bytes each)
    3. Number of Authority RRs/Number of additional RRs (4 bytes each)
    4. Questions (variable number) (8 bytes)
      • Name field with the name being queried
      • Type field with the kind of question
    5. Answers (variable number)
      • Resource Records which were queried
    6. Authority (variable number)
      • Records of other authoritative servers
    7. Additional Information (variable number)
      • Other helpful records (canonical hostnames and IPs, etc
  • DNS Vulnerabilities
    • DDoS attack
      • Flood server with messages
      • Many high-level servers have packet filters, preventing them from being directly attacked
    • Man-in-the-middle attack
      • Intercepts queries from hosts and returns bogus replies
      • May then redirect client to the real server so they don't know anything is wrong
